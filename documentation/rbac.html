Security & Permission Architecture

Nexus SMS protects your institutional data through a rigorous Role-Based Access Control (RBAC) model, ensuring every user has exactly the access they need—and nothing more.

User Tier Breakdown

👑

Super Administrator

Unrestricted access to the entire portal. Capable of modifying system core settings, license management, and master data purging.

💼

Principal / Manager

High-level oversight of all modules. Can view financial reports, staff performance, and global student metrics.

📝

Academic Staff

Permissions focused on classroom management: Attendance marking, mark entry, and student profile views.

💰

Accountant

Restricted to the Financial module. Handles fee collections, receipt generation, and ledger audits.

Security Feature

The 'RequireRole' Guard

Every internal API call in Nexus SMS is protected by a server-side guard. If a user tries to access a restricted function via URL manipulation, the system will instantly terminate the request and log a security violation.

Administrative Action If you notice a staff member has left the institution, deactivate their account immediately. Reactivation can only be performed by a Super Administrator.
