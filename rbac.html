Roles & Security

Understand how Nexus SMS protects your data with role-based access control, password security, and session management.

Access Control

Role-Based Access (RBAC)

Nexus SMS implements a three-tier role system. Each user is assigned one role that determines what modules they can access and what actions they can perform.

  • Admin: Complete access to all modules — Students, Staff, Fees, Grades, Attendance, Notifications, Export, and Settings
  • Teacher: Dashboard (class-specific), Attendance marking, Grade entry — limited to assigned classes
  • Accountant: Dashboard (finance-specific), Fees module, and Export — financial operations only

Dashboards

Role-Specific Dashboards

Each role sees a completely different dashboard tailored to their responsibilities:

  • Admin: Students, Staff, Classes, Fees, Attendance metrics + Grade Insights + Forecast + Student Growth
  • Teacher: My Classes count, My Students count, Attendance Status (Completed vs Pending), Pending Grades
  • Accountant: Total Collected, Total Due, Today's Collection, Overdue Students count

Password Policy

Password Security

Nexus SMS enforces strict password security:

  • Hashing: All passwords are hashed with SHA-256 and a unique random salt per user
  • Force Change: First-time and reset passwords require immediate change
  • Minimum Length: 6 characters minimum
  • Admin Reset: Admins can force-reset any staff password from the Staff module
Zero Plain-Text Storage Nexus SMS never stores plain-text passwords. Even administrators cannot view a user's current password.

Sessions

Session Security

User sessions are cryptographically secure with multiple layers of storage for reliability:

  • Duration: 8-hour sessions with automatic expiration
  • Storage: Dual-layer using Google Cache Service + Properties Service
  • Token Generation: Unique session IDs generated using crypto-secure methods
  • Logout: Sessions are destroyed on explicit logout
Reliability If the fast cache layer expires, Nexus falls back to the persistent Properties layer. Sessions survive cache flushes automatically.
